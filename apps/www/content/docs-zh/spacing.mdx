JetBrains UI 的间距首先服务于结构表达。它的重点不是“留白好看”，而是通过对齐、层级和留白，把依赖关系和分组关系讲清楚。

## 核心规则

- 能共用列对齐的独立控件，尽量放在同一条对齐线上。
- 用垂直和水平 inset 来表达控件之间的关系。
- 相关控件之间的距离应该明显小于不相关区块之间的距离。
- 能靠留白解决层级时，优先不要加额外边框或容器。
- 小规模相关内容可以用 group header；如果页面已经变得过大，就应该拆成更清晰的区块、标签页或 master-detail 结构。

## Figma 参考尺寸

| 场景                     | Figma 参考值                                                                          |
| ------------------------ | ------------------------------------------------------------------------------------- |
| 独立的带标签输入控件     | 控件块内部上下各 `6px` inset                                                          |
| 独立按钮或复选框行       | 控件块内部上下各 `8px` inset                                                          |
| Group header             | 标题内容上下各保留 `6px` 呼吸空间                                                     |
| 组、表格或列表的标题标签 | 标签下方保留 `6px` 间距                                                               |
| 成组输入框               | 相关行沿用同一套 `6px` 节奏                                                           |
| 成组复选框               | 相关行沿用同一套 `8px` 节奏                                                           |
| 带 hint 的控件           | 在主标签下为 hint 预留第二行空间                                                      |
| 对话框 footer            | 左右外边距 `20px`，帮助区和选项内容之间 `16px`，操作按钮之间 `12px`，按钮区上下 `8px` |

## 本仓库中的实现

间距 token 定义在 [tokens.css](/Users/mine/WebstormProjects/jetbrains-ui/apps/www/registry/styles/jetbrains/tokens.css)，并采用 `4px` 作为基础单位：

- `--space-1` = `4px`
- `--space-1_5` = `6px`
- `--space-2` = `8px`
- `--space-3` = `12px`
- `--space-4` = `16px`
- `--space-5` = `20px`

能选最近的 token 时，优先不要再引入新的任意像素值。

## 建议

- 先用对齐和间距解决结构问题，再考虑分割线。
- 紧凑控件内部优先复用 `6px` 和 `8px` 两套节奏。
- 把对话框 footer 当成独立布局区域处理，左右锚点要清晰。
- 尽量复用 token 化的间距值，不要把裸像素散落在组件里。
