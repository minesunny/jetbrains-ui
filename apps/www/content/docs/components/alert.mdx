## Overview

An alert informs users about the status of user or system initiated operations.
This component is built with Radix `AlertDialog` primitives, and built-in status icons are controlled by the `type` prop.

## Usage

```tsx
import {
  Alert,
  AlertAction,
  AlertActions,
  AlertCancel,
  AlertContent,
  AlertDescription,
  AlertFooter,
  AlertHeader,
  AlertTitle,
  AlertTrigger,
} from '@/components/jetbrains-ui/alert';
import { Button } from '@/components/jetbrains-ui/buttons';

<Alert>
  <AlertTrigger asChild>
    <Button variant="secondary">Open Alert</Button>
  </AlertTrigger>
  <AlertContent type="warning" size="auto" help check>
    <AlertHeader>
      <AlertTitle>Title</AlertTitle>
      <AlertDescription>Body text</AlertDescription>
    </AlertHeader>

    <AlertFooter>
      <AlertActions>
        <AlertCancel asChild>
          <Button variant="secondary">Cancel</Button>
        </AlertCancel>
        <AlertAction asChild>
          <Button>Review</Button>
        </AlertAction>
      </AlertActions>
    </AlertFooter>
  </AlertContent>
</Alert>;
```

## Type Switching

- Use `type` on `AlertContent` to switch built-in icon and semantic style:
  `info | error | warning | question`
- Use `help` on `AlertContent` (or demo props) to control the `QuestionMark` icon in `AlertFooter`.

## Sizing Rules

- Dialog default width follows Figma content guidance: `370px`.
- Use `size="default"` for fixed `370px` width.
- Use `size="wide"` for fixed `420px` width.
- Use `size="auto"` (default) for adaptive width (`w-fit` + `min-width: 370px`), and let long text or wider button groups expand the dialog naturally.
- Internal layout follows Figma paddings: `top 20 / right 20 / bottom 66 / left 60`.
- Height is content-driven: base content is about `150px`, and optional `check/help` content increases height as needed.
- Action and cancel buttons follow Button default size: `72x28`.
