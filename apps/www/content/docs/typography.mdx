JetBrains UI uses two typefaces:

- `Inter` for interface text
- `JetBrains Mono` for editor-like text and code-oriented surfaces

The official IntelliJ documentation also treats the UI font as a system setting. The default UI size is `13px`, and custom sizes should stay relative to that baseline instead of introducing arbitrary one-off values.

## Important Note About Weight

The Figma reference includes a note that all styles marked as `Medium` should be implemented as `Regular` because Figma and IntelliJ IDE rendering do not match exactly for that weight.

## UI Text Styles

| Style              | Typeface | Figma size / line height | Implementation weight | Typical use                                     |
| ------------------ | -------- | ------------------------ | --------------------- | ----------------------------------------------- |
| `H1`               | Inter    | `20 / 24`                | `Semibold`            | Page title, dialog title, major section heading |
| `H2`               | Inter    | `16 / 20`                | `Semibold`            | Section heading, panel heading                  |
| `Default`          | Inter    | `13 / 16`                | `Regular`             | Labels, inputs, trees, tables, standard UI text |
| `Default semibold` | Inter    | `13 / 16`                | `Semibold`            | Emphasized labels and compact headers           |
| `Paragraph`        | Inter    | `13 / 18`                | `Regular`             | Multiline descriptions and helper copy          |
| `Medium`           | Inter    | `12 / 16`                | `Regular`             | Dense secondary text and compact lists          |
| `Medium semibold`  | Inter    | `12 / 16`                | `Semibold`            | Compact subheads and emphasized small UI text   |

## Editor Text Styles

| Style          | Typeface       | Figma size / line height | Implementation weight | Typical use                                |
| -------------- | -------------- | ------------------------ | --------------------- | ------------------------------------------ |
| `Default`      | JetBrains Mono | `13 / 22`                | `Regular`             | Code blocks, inline code, editor-like text |
| `Default bold` | JetBrains Mono | `13 / 22`                | `Bold`                | Emphasized code and highlighted tokens     |
| `Small`        | JetBrains Mono | `12 / 22`                | `Regular`             | Line numbers and compact code metadata     |
| `Small bold`   | JetBrains Mono | `12 / 22`                | `Bold`                | Emphasized small editor text               |

## Implementation in This Repository

Typography tokens live in [tokens.css](/Users/mine/WebstormProjects/jetbrains-ui/apps/www/registry/styles/jetbrains/tokens.css):

- `--font-sans` for UI text
- `--font-mono` for editor text
- `--text-*` for font size tokens
- `--tracking-*` for letter spacing
- `text-ui-*` and `text-editor-*` utility classes for the canonical typography styles

Available utility classes:

| Style                 | Utility class              |
| --------------------- | -------------------------- |
| `H1`                  | `text-ui-h1`               |
| `H2`                  | `text-ui-h2`               |
| `Default`             | `text-ui-default`          |
| `Default semibold`    | `text-ui-default-semibold` |
| `Paragraph`           | `text-ui-paragraph`        |
| `Medium`              | `text-ui-medium`           |
| `Medium semibold`     | `text-ui-medium-semibold`  |
| `Editor Default`      | `text-editor-default`      |
| `Editor Default bold` | `text-editor-default-bold` |
| `Editor Small`        | `text-editor-small`        |
| `Editor Small bold`   | `text-editor-small-bold`   |

Typical usage:

```tsx
<h1 className="text-ui-h1">Settings</h1>
<p className="text-ui-paragraph">Helper text goes here.</p>
<code className="text-editor-default">const ready = true;</code>
```

## Recommendations

- Default to the UI font for components, forms, menus, and documentation copy.
- Reserve the editor font for code, code previews, and editor-adjacent surfaces.
- Prefer the documented size and line-height pairs over ad-hoc typography.
- If you need a custom size, treat it as a controlled delta from the default UI font rather than a new standalone scale.
