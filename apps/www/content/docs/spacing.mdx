JetBrains UI spacing is primarily structural. The goal is to show dependency, grouping, and hierarchy with alignment and whitespace before adding extra borders or containers.

## Core Rules

- Align independent controls to shared columns whenever possible.
- Use vertical and horizontal insets to show relationships between controls.
- Keep related controls visually closer to each other than to unrelated sections.
- Prefer whitespace over extra decoration. Add borders or filled containers only when spacing alone is not enough.
- Use group headers for smaller related sections. When a screen grows too large, split it into clearer sections, tabs, or a master-detail layout.

## Figma Reference Sizes

| Context                           | Reference from the Figma page                                                                                                                     |
| --------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| Separate labeled input            | `6px` top and bottom inset inside the control block                                                                                               |
| Separate button or checkbox row   | `8px` top and bottom inset inside the control block                                                                                               |
| Group header                      | `6px` breathing room above and below the header content                                                                                           |
| Label for a group, table, or list | `6px` gap below the label                                                                                                                         |
| Grouped text inputs               | Keep related rows on the same `6px` rhythm                                                                                                        |
| Grouped checkbox rows             | Keep related rows on the same `8px` rhythm                                                                                                        |
| Control with hint                 | Reserve a second line for the hint inside a taller control block                                                                                  |
| Dialog footer                     | `20px` outer horizontal inset, `16px` gap between help and option content, `12px` between action buttons, `8px` vertical inset around the buttons |

## Implementation in This Repository

Spacing tokens live in [tokens.css](/Users/mine/WebstormProjects/jetbrains-ui/apps/www/registry/styles/jetbrains/tokens.css) and use a `4px` base unit:

- `--space-1` = `4px`
- `--space-1_5` = `6px`
- `--space-2` = `8px`
- `--space-3` = `12px`
- `--space-4` = `16px`
- `--space-5` = `20px`

Use the nearest token instead of introducing arbitrary values whenever possible.

## Recommendations

- Start with alignment and spacing before reaching for dividers.
- Keep compact controls on consistent `6px` or `8px` internal rhythms.
- Treat dialog footers as a dedicated layout zone with clear left and right anchors.
- Reuse tokenized spacing values instead of scattering raw pixel literals across components.
